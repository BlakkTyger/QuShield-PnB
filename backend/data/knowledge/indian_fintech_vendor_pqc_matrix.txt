CORE BANKING SYSTEM (CBS) VENDORS

Infosys Finacle
PQC Status: IN PROGRESS
- Finacle 11 (latest): TLS 1.3 supported; PQC integration roadmap for Finacle 12 (2025-2026)
- Cryptographic dependencies: Uses Java Security (JCA/JCE) — BouncyCastle PQC in testing
- Key management: Integrates with Thales Luna HSMs (PQC-capable via firmware update)
- Assessment: MEDIUM readiness. Finacle is the most widely deployed CBS in Indian banking.
- Banking recommendation: Engage Infosys for Finacle PQC roadmap; target Finacle 12 upgrade for PQC readiness.
- Action items: Request formal PQC readiness statement from Infosys; negotiate SLA for PQC-capable release by 2026.

Tata Consultancy Services (TCS) BaNCS
PQC Status: LOW-MEDIUM
- BaNCS uses OpenSSL-based TLS; PQC not yet integrated in production releases
- TCS announced PQC research partnership with IBM in 2024
- Assessment: LOW-MEDIUM. No confirmed PQC release date.
- Banking recommendation: Escalate PQC requirements to TCS account manager; include PQC SLA in contract renewal.

Oracle FLEXCUBE
PQC Status: LOW
- FLEXCUBE Universal Banking uses Oracle's JDBC/SSL stack
- Oracle has PQC roadmap for Java Security and Oracle Database but FLEXCUBE-specific timeline unclear
- Assessment: LOW. No FLEXCUBE-specific PQC announcement as of 2024.
- Banking recommendation: Contact Oracle support for FLEXCUBE PQC roadmap; use network-layer PQC (load balancer termination) as interim mitigation.

Temenos Transact (formerly T24)
PQC Status: LOW-MEDIUM
- Uses TLS 1.3; internal cryptography via Java KeyStore (JKS)
- Temenos announced PQC assessment for Temenos Banking Cloud in 2024
- Assessment: LOW-MEDIUM.

FIS (Fiserv/Profile CBS)
PQC Status: LOW
- US-based CBS with Indian banking deployments; TLS 1.3 supported
- PQC roadmap announced for 2026-2027
- Assessment: LOW.

PAYMENT INFRASTRUCTURE

NPCI (National Payments Corporation of India)
PQC Status: PLANNING
- UPI: Digital signatures use ECDSA (quantum-vulnerable); NPCI issued advisory for banks to prepare PQC migration plan
- NACH (National Automated Clearing House): Uses RSA-2048 for mandate signing — quantum-vulnerable
- IMPS: HMAC-SHA256 for message authentication — quantum-safe (symmetric)
- BBPS (Bharat Bill Payment System): Classical TLS — migration required
- Assessment: Planning phase. NPCI has acknowledged quantum threat but no firm deployment dates.
- Banking recommendation: Participate in NPCI PQC working group; prepare for ECDSA → ML-DSA migration in UPI by 2026.

SWIFT (Society for Worldwide Interbank Financial Telecommunication)
PQC Status: IN PROGRESS
- SWIFT gpi (Global Payments Innovation): TLS 1.3 with classical cipher suites
- SWIFT CSP (Customer Security Programme): Mandates TLS 1.2+ (TLS 1.3 recommended from 2025)
- SWIFT published PQC position paper in 2024; migration guidance expected 2025
- Assessment: MEDIUM. SWIFT is moving deliberately; coordination with 200+ countries requires global consensus.
- Banking recommendation: Monitor SWIFT PQC announcements; maintain SWIFT connector compatibility while planning local PQC.

Razorpay
PQC Status: LOW
- Payment gateway using classical TLS 1.3 with ECDHE key exchange
- No public PQC roadmap announced
- Banking recommendation: Treat Razorpay integration as a third-party quantum risk; classify payment callbacks as medium-priority for HNDL risk assessment.

PayU India
PQC Status: LOW
- Classical TLS; no PQC announcements
- Assessment: LOW.

TECHNOLOGY VENDORS COMMON IN INDIAN BANKING

IBM (Middleware, MQ, Z Series)
PQC Status: HIGH
- IBM Quantum Safe Cryptography: z16 mainframes support PQC natively
- IBM MQ: TLS PQC support in MQ 9.3+
- IBM Cloud HSM: PQC key management available
- Assessment: HIGH. IBM is a leader in enterprise PQC deployment.
- Banking recommendation: IBM MQ users can upgrade to 9.3+ for PQC-capable messaging bus.

Cisco (Networking)
PQC Status: HIGH
- IOS-XE 17.14+: Hybrid PQC key exchange for VPN tunnels (ML-KEM + ECDH)
- Cisco Firepower NGFW: PQC-aware inspection (pass-through for PQC traffic)
- Cisco SD-WAN: PQC key exchange in roadmap for 2025
- Assessment: HIGH for networking; medium for security appliances.
- Banking recommendation: Upgrade core routers/switches to IOS-XE 17.14+ for PQC VPN connectivity.

Palo Alto Networks (Firewall/NGFW)
PQC Status: MEDIUM
- PAN-OS 11.0+: Can pass through PQC TLS traffic (no deep inspection support yet)
- SSL/TLS decryption: Cannot decrypt PQC-only sessions (no ML-KEM support in decryption engine)
- Assessment: MEDIUM. Inspect-and-decrypt workflows will break for PQC sessions until vendor update.
- Banking recommendation: Plan firewall inspection policy updates when deploying PQC TLS.

Fortinet (FortiGate)
PQC Status: MEDIUM-LOW
- FortiOS 7.4: TLS 1.3 pass-through; no PQC-aware deep inspection
- Assessment: MEDIUM-LOW.

QUANTUM RISK SUMMARY FOR INDIAN BANKING ECOSYSTEM
High Risk (Immediate Action): NPCI UPI ECDSA signatures, NACH RSA mandate signing, SWIFT classical key exchange
Medium Risk (Plan within 18 months): Core banking system TLS, payment gateway integrations, Finacle/BaNCS CBS
Lower Risk (Plan within 36 months): Internal management systems, logging, monitoring infrastructure
Already Quantum-Safe: HMAC-SHA256 based MACs (IMPS), AES-256 encrypted data at rest, SHA-256 hashed data
