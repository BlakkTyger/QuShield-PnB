Cloud Provider and Database Vendor PQC Readiness Status (2024-2025)
Source: Vendor public documentation, security blogs, and published roadmaps
Tags: vendor, aws, azure, gcp, mongodb, cloud, pqc, readiness, third_party

AWS (Amazon Web Services)
PQC Status: IN PROGRESS
- AWS KMS: Supports RSA-4096, ECC P-384 for key management; ML-KEM and ML-DSA in preview (2025)
- AWS Certificate Manager (ACM): Planning hybrid PQC certificates; expected 2025-2026
- AWS Load Balancer / CloudFront: X25519MLKEM768 hybrid TLS supported in CloudFront (announced 2024)
- AWS IoT: PQC-safe TLS in roadmap for 2026
- S3/Storage: Server-side encryption uses AES-256 (quantum-safe); key wrap uses RSA (vulnerable)
- Assessment: MEDIUM readiness. Internet-edge services progressing well; key management infrastructure lags.
- Banking recommendation: Use CloudFront for PQC-capable CDN edge. Migrate KMS key wrapping to ML-KEM when available.

Microsoft Azure
PQC Status: IN PROGRESS
- Azure Key Vault / Managed HSM: Thales Luna HSM backend; PQC key types planned for 2025
- Azure Certificate Manager: Planning hybrid PQC certificate issuance (no firm date announced)
- Azure Front Door / Application Gateway: TLS termination; PQC support roadmap in progress
- Azure Active Directory: Authentication protocols (ECDSA JWT signing); PQC migration announced 2025
- Azure SQL / Cosmos DB: TDE (Transparent Data Encryption) uses AES-256 (quantum-safe); TLS to DB uses classical
- Assessment: MEDIUM-LOW readiness. Strong enterprise commitment but implementation lags AWS.
- Banking recommendation: Monitor Azure HSM PQC availability closely; plan for 2026 deployment.

Google Cloud Platform (GCP)
PQC Status: LEADING
- Cloud Load Balancing: X25519MLKEM768 hybrid TLS enabled by default (via BoringSSL), production since 2024
- Google Certificate Authority Service: Hybrid PQC certificates in preview
- Cloud KMS / Cloud HSM: ML-KEM key generation available in limited preview (2025)
- BigQuery / Cloud Storage: AES-256 encryption (quantum-safe symmetric); key management still classical
- Google Workspace: Gmail, Drive use classical TLS but GCP network uses hybrid PQC internally
- Assessment: HIGH readiness. Google has the most mature PQC deployment among hyperscalers.
- Banking recommendation: GCP is currently the most PQC-ready cloud provider for TLS workloads.

MongoDB
PQC Status: LOW (planning phase)
- MongoDB Atlas (managed): TLS 1.3 supported; cipher suites still classical (ECDHE-RSA-AES256-GCM-SHA384)
- MongoDB Community/Enterprise: TLS via OpenSSL — PQC depends on OpenSSL version deployed
- Driver support: MongoDB drivers (Java, Python, Node.js) do not yet support PQC cipher suites
- Atlas Search (Lucene-based): No cryptographic exposure — uses internal plaintext indexing
- Replica set communication: TLS 1.3 but classical key exchange
- Assessment: NOT PQC-ready. No announced timeline for PQC support in MongoDB Atlas.
- Banking recommendation: MongoDB connections from banking applications are quantum-vulnerable for key exchange. Priority: HIGH. Mitigation: ensure MongoDB is behind VPC with no internet exposure; plan migration when driver support becomes available (estimated 2026-2027).
- Risk: MongoDB Atlas public endpoints (if used) are exposed to HNDL attacks.

PostgreSQL
PQC Status: LOW (community library dependent)
- PostgreSQL uses OpenSSL for TLS; PQC support depends on OpenSSL 3.4+ with OQS provider
- No native PQC support built into PostgreSQL core
- pgcrypto extension: Uses classical RSA and ECDSA — NOT quantum-safe
- SSL/TLS configuration: Can be upgraded to hybrid PQC if host OpenSSL supports it
- Assessment: PARTIAL. TLS layer can be made PQC-compatible; application-layer pgcrypto remains classical.
- Banking recommendation: Upgrade OS OpenSSL to 3.4+, configure hybrid TLS for PostgreSQL connections. Replace pgcrypto RSA calls with application-layer ML-DSA.

Redis
PQC Status: VERY LOW
- Redis TLS uses classical cipher suites
- Redis Cluster: Internal node communication uses classical TLS
- No PQC roadmap announced
- Assessment: NOT PQC-ready.
- Banking recommendation: Redis used for session tokens and caching — HNDL risk is lower (data is short-lived). Medium priority for PQC migration.

Nginx / Apache HTTP Server
PQC Status: PARTIAL (OpenSSL dependent)
- Nginx with OpenSSL 3.4+ and liboqs: Supports hybrid PQC TLS
- Apache httpd with OpenSSL 3.4+: Same capability
- Configuration: Requires ssl_protocols and ssl_ciphers update
- Banking recommendation: Update underlying OpenSSL, then configure X25519MLKEM768 cipher group. This is the fastest PQC win for externally-facing web servers.

Cloudflare (CDN/WAF/DNS)
PQC Status: HIGH (PRODUCTION)
- X25519MLKEM768 enabled by default for all Cloudflare zones (2024)
- Cloudflare CA: Issuing certificates with plans for hybrid signing
- Cloudflare Tunnel: Uses QUIC+TLS 1.3 with hybrid PQC key exchange
- Assessment: HIGH readiness. Cloudflare is the most PQC-ready CDN/WAF vendor.
- Banking recommendation: Route all internet-facing banking APIs through Cloudflare for immediate PQC key exchange protection.

Akamai
PQC Status: MEDIUM
- TLS 1.3 supported on all edge nodes
- Hybrid PQC key exchange (X25519MLKEM768) in beta for enterprise customers (2024-2025)
- Assessment: MEDIUM. Enterprise customers can enable hybrid PQC on request.

F5 Networks (Load Balancers, WAF)
PQC Status: LOW-MEDIUM
- BIG-IP: TLS 1.3 with classical cipher suites; PQC roadmap for 2025-2026 firmware
- F5 Distributed Cloud: Hybrid PQC in development
- Assessment: MEDIUM. Plan for firmware upgrade cycle aligned with PQC availability.

Thales (HSM Vendor)
PQC Status: HIGH
- Luna Network HSM 7: PQC algorithms supported (ML-KEM, ML-DSA) via firmware 7.8.0+
- Assessment: HIGH. Leading HSM vendor for PQC. Most banks using Thales HSMs can deploy PQC with firmware update.

Utimaco (HSM Vendor)
PQC Status: HIGH
- Atalla AT1000 and SecurityServer HSMs support PQC via Q-Safe plugin
- Assessment: HIGH. PQC-capable with software update.

Entrust (CA/PKI)
PQC Status: MEDIUM-HIGH
- nShield HSMs: PQC capabilities in development
- Certificate Services: Hybrid PQC certificates available in 2025
- Assessment: MEDIUM-HIGH. Major CA with committed PQC roadmap.

DigiCert (CA)
PQC Status: HIGH (industry leader)
- DigiCert Trust Lifecycle Manager: Full hybrid PQC certificate issuance (ML-DSA-65 + RSA-2048 dual-signed)
- Banking recommendation: DigiCert is the recommended CA for Indian banks beginning PQC certificate migration.
